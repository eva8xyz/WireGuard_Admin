# WireGuard 統合管理コンソール (WireGuard Unified Portal console)

> 🌐 **Looking for English documentation?** Please check [README.md](./README.md) for the English version.

本システムは、実稼働の WireGuard (WG) サーバーを完全リアルタイムで監視・制御、接続ピア(Peer)の一括プロビジョニング、および動的なクライアント向け `.conf` 設定自動生成を行う、完全日本語対応の実用グレードの管理ポータルです。

## ✨ 特徴
1. **実稼働リアルタイム同期**: 
   - サーバー上の `wg show dump` コマンドを2秒ごとに解析し、各端末の転送通信量（Rx/Tx）、現在の通信レート、および最新のハンドシェイク時刻をリアルタイムに自動更新します。
2. **包括的セットアップウィザード**:
   - 初回起動時、データベースが未構築の場合には、管理者IDとパスワードの作成、インターフェース名、ゲートウェイCIDR、外部ドメインEndpoint、DNS、MTU、キープアライブ設定、および暗号鍵ペアの紐付けを行う美しく堅牢なインストーラーから自動的に開始されます。
3. **柔軟なピア(Peer)自動 & 既存インポート管理**:
   - 暗号キーの自動自動生成付き「新規ピア登録」、既存クライアントの認証鍵を持ち込む「手動既存ピア登録」、既存 `.conf` から一括パースして登録する「一括同期インポート（複数ピア対応）」に完全対応。
   - 「役職」や「メールアドレス」の不要な管理項目を排除し、シンプルな端末/ユーザー名とデバイス名の2点管理に最適化されています。
4. **自動実行 & ローカルサンドボックスフォールバック**:
   - Dockerなどのサンドボックス環境等、WireGuardカーネルドライバや `wg` コマンドが未検出の環境においてもクラッシュせず、高精度な高機能JSONデータベースモードに完全自動フォールバック。

## 🛠️ 事前準備と手動設定事項（実サーバー運用時）

本アプリケーションを実用のサーバー上で稼働させ、WireGuardに設定をリアルタイム反映するための手動設定事項は以下の通りです。

### 1. `wg` コマンドに対する `sudo` 実行権限の付与（セキュリティ設定）
Node.jsサーバーを実行するシステムユーザー（例: `node` や `ubuntu`）に対して、非対話的に `wg` コマンドを実行できるように `sudoers` を設定します。

ターミナルで `sudo visudo` を実行し、末尾に以下の設定を追加してください。

```bash
# Node.js を実行するシステムユーザーに wg コマンドのパスワード不要権限を付与
<実行ユーザー名> ALL=(ALL) NOPASSWD: /usr/bin/wg, /usr/sbin/wg
```
※ `/usr/bin/wg` のパスは環境により異なる場合があるため、`which wg` コマンドでパスを確認して適切に反映してください。

### 2. ポートの解放
WireGuardのUDP待受ポート（デフォルトは `51820`）および本ウェブUIのリスニングポート（`3000`）について、ルーターおよびLinuxファイアウォール（`ufw` や `iptables`）で許可を設定してください。

```bash
# 例: UFW を使用した設定
sudo ufw allow 51820/udp
sudo ufw allow 3000/tcp
```

## 🚀 起動方法とコマンド

### 1. Windows 11 Pro 自動セットアップ・起動 (一クリック簡単起動)
Windows 11 Pro 環境向けに、管理者権限への自動昇格（UAC）およびビルド、実行を自動化する **Batchスクリプト** を追加しました。

*   **初回セットアップ & 起動 (`setup_start.bat`)**:
    - ダブルクリックすると、自動的に「管理者として実行」の許可（UACプロンプト）を要求します。
    - パッケージの依存関係をクリーンインストール (`npm install`) した後、本番用ビルドを行い、自動的にサーバーを本番モード (`NODE_ENV=production`) で起動します。
    - **E:\\WireGuard Admin\\wireguard-vpn-portal** に配置した状態でそのまま実行可能です。
*   **2回目以降の高速起動 (`start.bat`)**:
    - 初回インストール済みの状態で、ビルドと本番起動のみを瞬時に行いたい場合に使用します。こちらもダブルクリックするだけで管理者昇格から自動起動まで完了します。

---

### 2. 一般的なコマンドライン起動 (Windows / Linux / macOS)
手動でコマンドを実行して起動する場合は以下の手順に従ってください。

```bash
# 1. 依存関係のクリーンインストール
npm install

# 2. 開発環境でのブート (サーバー & クライアント一体型)
# tsx を使用して TypeScript サーバーを 0.0.0.0:3000 にて起動します
npm run dev

# 3. 本番環境向けビルドと実行
# フロントエンドを Vite で最適化し、サーバーを esbuild を用いて dist/server.cjs にコンパイルします
npm run build

# 本番起動
npm start
```

## 🌐 多言語・初期設定時の言語仕様
本システムは、**英語（EN）**および**日本語（JP）**に完全対応しています。

*   **デフォルトの言語**: 初期状態またはリセット直後のインストールウィザードでは、**英語（EN）**が標準言語としてロードされます。
*   **日本語（JP）への切り替え**: 
    - セットアップ画面およびログイン画面の右上にある **「🌐 日本語に切り替える (JP)」** ボタンから瞬時に日本語画面へ切り替えることができます。
    - ログイン後の管理者用ダッシュボードのヘッダー右上にも **「🌐 JP」** または **「🌐 EN」** ボタンが配置されており、いつでも自由に表示言語を切り替えることができます。選択した言語はブラウザに記憶されます。

## 💾 データ永続化およびシステム設定ファイル

本システムは、データベースや設定値の管理をすべてサーバー内の安全なローカルJSONファイルにて直接永続化します。これにより、外部DBなしでポータブル。かつ迅速なバックアップと復元を可能にしています。

*   `wg-portal-config.json`: 管理者認証用ハッシュ、DNS、MTU、Endpoint、インターフェース定義、および暗号鍵などのコアシステム設定。
*   `wg-portal-peers.json`: 登録されたすべてのピア(User)のアクセス情報。

### システム設定の完全初期化（初期インストール状態へ戻す）
管理者ログイン後、ヘッダーに配置されている **「システム設定・DBの完全初期化（セットアップに戻る）」** ボタンをクリックすると、ファイルが正常に安全削除され、即座に「初期セットアップウィザード」に画面がリダイレクトされます。既存データの全面移行や、別のサーバーにおける再構築の際にご活用ください。
